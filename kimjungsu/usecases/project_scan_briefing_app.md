# Use Case: Daily Briefing App Monorepo

## 상황

이 유즈케이스의 서비스는 단순한 개인 메모 앱이 아니다.
대상 고객은 **10~80명 규모 스타트업의 팀 리드 또는 운영 리드**다.

이 고객은 아침마다 다음을 동시에 처리해야 한다.

- 오늘 일정 확인
- 중요한 미팅 준비 포인트 파악
- 팀에서 어제 생긴 이슈 확인
- 오늘 먼저 처리할 우선순위 결정

현재 상태는 보통 이렇다.

- Google Calendar를 열고
- Slack을 확인하고
- Notion이나 개인 메모를 다시 보고
- 작업 목록을 따로 열어보며
- 머릿속으로 오늘의 순서를 재구성한다

즉 고객의 진짜 문제는 "정보가 없다"가 아니라
**흩어진 정보를 아침 2분 안에 행동 순서로 정리하지 못한다**는 점이다.

이 서비스의 목표는 아래와 같다.

- 하루 시작 전에 오늘의 브리핑을 2분 안에 이해하게 한다
- 중요한 일정 준비를 놓치지 않게 한다
- 팀 상황과 개인 우선순위를 한 화면에서 연결한다

당장 필요한 것은 "전체를 다 읽는 것"이 아니라,
**어떤 영역과 파일부터 읽어야 이 서비스의 고객 가치가 어디서 만들어지는지 빨리 이해할 수 있는지** 결정하는 것이다.

실험 입력은 [`fixtures/briefing_app`](../fixtures/briefing_app/README.md)이다.

## 의사결정 질문

1. 이 저장소는 단일 앱인가, 문서/프론트/백엔드가 섞인 혼합형 저장소인가?
2. 프론트엔드와 백엔드 anchor는 각각 어디에 있는가?
3. 상위 디렉토리 중 어디부터 보는 것이 가장 빠른가?
4. 지금 바로 열 파일 5개는 무엇인가?

## Stage 0: Service Lens

### 사용자

- 아침마다 여러 도구를 넘나들며 오늘의 우선순위를 정리해야 하는 팀 리드 / 운영 리드

### 핵심 Job

- "오늘 무엇을 먼저 해야 하는지 2분 안에 파악하고 바로 행동에 들어가고 싶다"

### 가치 순간

- 앱을 열었을 때 일정, 준비 포인트, 우선순위가 한 번에 정리되어 바로 행동으로 이어지는 순간

### 기술의 역할

- 웹 앱은 사용자가 가치를 체감하는 접점이다
- Python 서비스는 브리핑 생성 품질과 우선순위 로직을 담당하는 내부 도구다
- 문서는 전체 방향과 제품 의도를 빠르게 복원하는 도구다

### 지금 보지 않아도 되는 것

- 세부 구현 최적화
- 프레임워크 취향
- 내부 구조의 미세한 코드 스타일

즉 이 유즈케이스에서 기술은 "무엇을 만들었는가"보다
"사용자가 어디서 가치를 느끼는가"를 복원하기 위한 도구로 사용된다.

## 단계별 실험 내용

### Stage 1

- 목적: 저장소 전체 형태를 본다
- 보고 싶은 신호: markdown, python, typescript가 함께 존재하는지
- 기대 해석: 단일 언어 저장소가 아니라 혼합형 저장소다
- 서비스 연결: 브리핑 가치가 문서, 고객 접점, 생성 로직에 걸쳐 있음을 추정한다

### Stage 2

- 목적: 프론트/백엔드 anchor를 찾는다
- 보고 싶은 신호: `apps/web/package.json`, `apps/web/tsconfig.json`, `services/briefing/pyproject.toml`
- 기대 해석: 웹 앱과 Python 서비스가 분리된 구조다
- 서비스 연결: 사용자가 보는 접점과 브리핑 품질을 만드는 로직이 분리되어 있음을 본다

### Stage 3

- 목적: 어떤 영역부터 파야 하는지 정한다
- 보고 싶은 신호: `apps`가 1위, `services`가 2위
- 기대 해석: 첫 탐색은 사용자-facing web 영역부터 시작하고, 구현 상세는 서비스로 내려간다
- 서비스 연결: 가치 접점이 앱에 있으므로 앱을 먼저 보고, 품질 로직은 그 다음에 본다

### Stage 4

- 목적: 바로 열 파일을 정한다
- 보고 싶은 신호: 루트 README, app README, service README, app manifest가 상위 추천
- 기대 해석: 온보딩 순서가 `전체 설명 -> 고객 접점 설명 -> 가치 엔진 설명 -> 실행/설정 정보`로 흘러간다
- 서비스 연결: 서비스 이해 순서가 기술 순서가 아니라 가치 복원 순서로 정렬된다
